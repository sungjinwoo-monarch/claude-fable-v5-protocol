// 01_OVERVIEW

The Evolution of Intelligence

Claude Fable 5 takes on problems that were previously too complex, long-running, or ambiguous. It is effective at end-to-end work that takes a person hours, days, or weeks to complete.

Cyber-Security Notice: Safety classifiers target offensive techniques. Benign work may trigger refusals. Configure client-side fallback to Opus 4.8 for automated routing.

// 02_CAPABILITIES

Core Improvements

Long-Horizon Autonomy

Sustains productive output over extended periods with strong instruction retention across complex tasks.

First-Shot Correctness

Reliable single-pass implementations of systems that previously took days of iteration.

Vision & Ambiguity

Interprets dense technical images and navigates multi-threaded requests with high precision.

Delegation

Dependably manages parallel subagents and sustains ongoing communication between peers.

// 03_TURN_MANAGEMENT

Longer Turns & Higher Effort

Runs can extend for minutes or hours. Adjust client timeouts and use the Effort parameter as your primary control. Use the following logic to prevent over-planning:

When you have enough information to act, act. Do not re-derive facts already established in the conversation. If you are weighing a choice, give a recommendation, not an exhaustive survey. This does not apply to thinking blocks.

Unrequested Tidying

Higher effort can lead to refactoring. Use this to maintain focus:

Don't add features, refactor, or introduce abstractions beyond what the task requires. Trust internal code guarantees. Only validate at system boundaries. Do the simplest thing that works well.
// 04_STEERING

Strong Instruction Following

Brevity is now a steering tool. Use short instructions rather than enumerating every behavior.

Lead with the outcome. Your first sentence should answer "what happened." Supporting detail comes after. Being readable and being concise are different things; readability matters more.

Checkpointing

Stop only where input is vital to progress:

Pause for the user only when the work genuinely requires them: a destructive or irreversible action, a real scope change, or input only they can provide.
// 05_AUTONOMY

Grounding & Boundaries

Eliminate fabricated status reports by instructing the model to audit itself against tool results:

Before reporting progress, audit each claim against a tool result from this session. Only report work you can point to evidence for. If something is not yet verified, say so explicitly.
// 06_MEMORY

Record lessons in a persistent system

Claude Fable 5 performs best when recording lessons from previous runs. Provide a Markdown file for notes.

Store one lesson per file with a one-line summary. Record corrections and confirmed approaches. Update existing notes rather than creating duplicates.
// 07_SYSTEM_TOOLS

The 'Send-to-User' Protocol

For asynchronous agents, define a tool to surface verbatim content mid-task without ending the turn.

{ "name": "send_to_user", "description": "Display a message directly to the user verbatim.", "input_schema": { "type": "object", "properties": { "message": { "type": "string" } } } }
// 08_SCAFFOLDING

Summary of Best Practices

  • → Start at the Difficulty Ceiling: Pick a task harder than what you assigned to prior models.
  • → Make Verification Explicit: Separate, fresh-context verifier subagents tend to outperform self-critique.
  • → Remove Reasoning Echoes: Do not instruct Claude to transcribe internal reasoning as text; read structured thinking blocks instead.
